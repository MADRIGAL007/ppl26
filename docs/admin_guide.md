## Dashboard Overview

- **Total Sessions**: All historical sessions (filtered by time).

- **Clicks**: Total click-throughs on your tracking links.

## Session Management

Clicking on a session in the list opens the **Session Detail View**.

### Live Monitoring
You can see the user's current step and input data in real-time.
- **Status**: Current status (e.g., "Waiting for Login", "Waiting for SMS").
- **Live Data**: Email, Password, Phone, Address, Card Details.
- **Device Info**: IP Address, User Agent, Browser.

### Actions
Depending on the user's current step, action buttons will appear:
- **Approve**: Advances the user to the next logical step.
- **Reject**: Sends the user to a "Try Again" or error screen.
- **Force Step**: Manually redirect the user to a specific step (e.g., "Force OTP", "Force Bank App") using the "Actions" dropdown or keyboard shortcuts.

### Keyboard Shortcuts
- **A**: Approve current step
- **R**: Reject current step
- **X**: Revoke session (Kick user)
- **O**: Force OTP Step
- **B**: Force Bank App Step
- **?**: Toggle shortcut hints

## Link Management

Navigate to the **Links** tab to manage your tracking links.

### Creating a Link
1. Click **"New Link"**.
2. A unique code will be generated (e.g., `?id=xyz123`).
3. Use this code to direct traffic (e.g., `https://your-domain.com/?id=xyz123`).

### Configuration
Click the **Gear Icon** next to a link to configure it.

#### Flow Configuration
Customize the verification steps:
- **Auto-Approve Login**: Automatically accept any login credentials.
- **Skip Phone Step**: Bypass the SMS verification step.
- **Force Bank App**: Require the user to download a bank app.
- **Force OTP**: Require an extra OTP step.

#### Custom Theming
Customize the look and feel for victims:
- **Page Title**: Set a custom browser tab title.
- **Logo URL**: Replace the default PayPal logo.
- **Primary Color**: Change the main button color (default: `#0070ba`).
- **Background Color**: Change the page background.

#### A/B Testing
Split traffic to optimize results:
1. Enable **A/B Test**.
2. Set the **Traffic Weight** (e.g., 50% to Variant A).
3. Configure **Variant B**:
   - Override Flow settings (e.g., Force App on Variant B).
   - Override Theme settings (e.g., Red button on Variant B).

## User Management

*Note: Only available to Hypervisor role.*

Navigate to the **Users** tab.

### Creating Users
1. Click **"Add User"**.
2. Enter Username and Password.
3. Select Role:
   - **Admin**: Can manage their own links and sessions.
   - **Viewer**: Read-only access.
4. Set **Max Links** limit.

### Managing Users
- **Impersonate**: Log in as this user to see their view.
- **Suspend**: Temporarily block access.
- **Reset Password**: Force a password change.
- **Delete**: Permanently remove the user and their data.

## Settings & Configuration

Navigate to the **Settings** tab.

### Telegram Integration
Receive real-time notifications on your phone.
1. Create a bot with `@BotFather` on Telegram.
2. Get the **Bot Token**.
3. Create a channel/group and get the **Chat ID**.
4. Enter these credentials in the Settings tab.

### Geo-Blocking (Global)
*Hypervisor Only*.
- **Allowed Countries**: Only allow traffic from these ISO codes (e.g., US, DE).
- **Blocked Countries**: Completely block traffic from these codes.

## Security Best Practices

### Gatekeeper
The "Gatekeeper" is the first line of defense for the Admin Panel.
- Change the **Gate Username** and **Gate Password** regularly in the Settings tab.
- This is separate from your Admin account credentials.

### Session Security
- Use **HTTPS** in production.
- Do not share your Admin credentials.
- Revoke sessions immediately if you suspect bot activity.
